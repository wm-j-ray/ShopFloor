# Custom Framework Builder

> **Purpose:** Allows a writer to define their own structural beat sequence for use as a named framework in the Beat Registry and Conformance Report.
>
> **Instance naming:** When creating an instance, save as `[ID]_Name.md`, set `profile_version` and `date_instantiated`.

> 🟡 **TIER 2 — Important.** Complete after core Tier 1 structures are established.

## Beat Definitions

| Beat ID | Beat Label | Definition | Required [yes | no] | Act | Target Position [% of manuscript] | Grading Weight [1-3] |
|---|---|---|---|---|---|---|---|

One entry per beat. All fields required for grading to function.
Beat IDs must follow the standard format [BT-001, BT-002, ...] for compatibility with Beat Registry.

## Grading Weight Key

1 = minor beat — absence noted but not penalized heavily
2 = significant beat — absence weakens conformance score
3 = essential beat — absence critically reduces conformance score
